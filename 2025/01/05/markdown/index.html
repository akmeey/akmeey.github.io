Markdown

发表于 2025-01-05 分类于 Blog ， Markdown Waline：

1. 简介

Markdown是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档，然后将其转换成有效的XHTML（或者HTML）文档。因此，Markdown 有两个含义

  1. 纯文本格式化语法
  2. 将纯文本格式转换为 HTML 的工具

Markdown 的格式化语法只是用来解决如何使用纯文本来表达这个问题的。
Markdown 的语法非常小，它仅对应于 HTML 标签的一个非常小的子集，所以，Markdown 并不是 HTML 的替代品。 HTML 是一种发布格式，Markdown 是一种写作格式。
由于 Markdown 的轻量化、易读易写特性，并且对于图片，图表、数学式都有支持，目前许多网站都使用 Markdown 来撰写帮助文档或是用于在论坛上发表消息。

2. 语法

2.1 内联 HTML

对于 Markdown 语法未涵盖的任何标记，都可以使用 HTML 标签来实现。

  • 对于行级 HTML 元素，例如 <span> 、 <cite> 或 <del>，可以在 Markdown 段落、列表项或标题中的任何位置使用。
  • 对于块级 HTML 元素，例如 <div> ， <table> 、 <pre> 、 <p> 等，必须用空行与周围内容分隔，并且块的开始和结束标记不需要使用制表符或空格进行缩进。

2.2 特殊字符的自动转义

在 HTML 中，有两个字符需要特殊处理： < 和 &。左尖括号 < 用于开始标签，与符号 & 用于表示 HTML 实体。
若要将 < 和 & 用作文字字符，则必须对它们进行转义，< 转义后就是 &lt;，& 转义后就是 &amp;。
Markdown 会自动为 < 和 & 字符进行转义操作

  • 若使用 < 作为 HTML 标签的分隔符，Markdown 会将其视为内联 HTML ，< 保持不变，否则 < 会被转义为 &lt;。
  • 若使用 & 作为 HTML 标签的一部分，则 & 保持不变，否则 & 会被转义为 &amp;。

在 Markdown 的行内代码和代码块中，< 和 & 符号总是会被自动转义的。这使得使用 Markdown 编写 HTML 代码变得很容易。

2.3 段落和换行符

段落是由一个或多个空行分隔的一行或多行连续文本，这里的一行或多行连续文本的含义是 Markdown 支持“硬包装” 文本段落。
因为 Markdown 不会将每个换行符转换为 <br /> 标记，所以，当您想使用 Markdown 插入 <br /> 标记时，可以用两个或多个空格结束一行，然后键入回车。即行末的两个空格会产生断行。例如：
登鹳雀楼
王之涣〔唐代〕
白日依山尽，黄河入海流。
欲穷千里目，更上一层楼。

Markdown 中可以使用 > 来引用文本。
例如：

会当凌绝顶，
一览众山小。

引用也是可以嵌套的，例如：

题乌江亭
杜牧〔唐代〕

胜败兵家事不期，包羞忍耻是男儿。
江东子弟多才俊，卷土重来未可知。

在引用中也是可以使用 Markdown 语法的，例如：

This is a ERROR.

  2. This is the second list item.

2.7 代码块

Markdown 将代码块包装在 <pre> 和 <code> 标记中。
要在 Markdown 中生成代码块，只需将代码块的每一行缩进至少 4 个空格或 1 个制表符即可。 Hexo 中的 Markdown 引擎不支持该语法，所以必须使用 Github 中的代码块语法。
Github 的 Markdown 语法（GFM）是使用 三反引号 ``` 来标注代码块的。

在代码块内，与符号 & 和尖括号 < 和 > 会被自动转换为 HTML 实体。这使得使用 Markdown 包含示例 HTML 源代码变得非常容易，只需粘贴并缩进即可。

2.9 文本样式

Markdown 将星号 * 和下划线 _ 视为强调指示符。

  • 用 * 或 _ 包裹的文本将用 HTML <em> 标记包裹。
  • 用 ** 或 __ 包裹的文本将用 HTML <strong> 标记包裹。

若使用空格包围 * 或 _，则 * 或 _ 会被视为字面星号或下划线。
若要在原本用作强调分隔符的位置生成字面星号或下划线，则可以使用反斜杠转义它。
